# Claude Code Book 双语站点与双语仓库首页设计

**目标**

为当前网站和 GitHub 仓库首页增加中英双语内容体系，并以英文作为默认展示语言。

**核心决策**

- 网站默认英文
- 独立双语页面，不做运行时翻译
- 英文默认页使用现有主路径：`index.html`、`book.html`
- 中文镜像页使用：`index.zh.html`、`book.zh.html`
- GitHub 仓库首页默认英文：`README.md`
- 中文仓库介绍页：`README.zh-CN.md`
- 英文版为完整正文版，覆盖 16 章正文与附录，而不是仅导航与摘要翻译

**为什么采用独立双语页面**

当前站点是纯静态 HTML，且 `book.html` 是长篇内容页。独立页面比单页切换更稳，更容易控制长文本排版，不会让中英文两套内容在同一个 DOM 里互相干扰，也更适合 GitHub Pages 的静态托管方式。

**网站信息架构**

- `index.html`: 英文首页
- `book.html`: 英文在线阅读页
- `index.zh.html`: 中文首页
- `book.zh.html`: 中文在线阅读页
- 顶部导航、首屏按钮、章节目录、侧边栏、页脚说明均提供中英切换入口
- 中英页面使用同一套样式系统，但通过 `html[lang="en"]` 做英文定制排版

**GitHub 首页双语方案**

GitHub 只能自动渲染 `README.md` 作为仓库首页，因此默认英文应由 `README.md` 承担。中文介绍页单独放在 `README.zh-CN.md`，并在两个 README 顶部互相链接，实现“英文默认、中文可切换”的仓库首页体系。

**英文排版约束**

为了避免 `book.html` 英文版出现文字出框问题，英文页应采取以下策略：

- 长标题和长摘要使用原生排版，不使用 `pretext` 装饰性断行
- 英文导航、侧边栏目录、按钮文案允许换行
- 英文长词启用 `overflow-wrap: anywhere` 与 `hyphens: auto`
- 英文章节标题、章节摘要、目录项、阅读路径卡片在 `html[lang="en"]` 下单独调字号、行高和列宽
- 代码块、表格、引用块继续使用现有滚动和容器规则，避免横向溢出

**实施范围**

1. 创建英文版首页与英文版正文页
2. 创建中文版镜像页并接入中英切换导航
3. 创建英文版 Markdown 书稿以匹配英文阅读页
4. 重写 `README.md` 为英文版，新增 `README.zh-CN.md`
5. 增加英文字体排版保护规则，特别是 `book.html`
6. 增加基础测试与构建验证

**风险**

- `book.html` 为长篇静态文件，中英文双份维护成本较高
- 英文版目录和按钮文案更长，必须重点检查 `book.html` 的侧边栏与首屏布局
- GitHub Pages 可能因静态资源缓存导致切换样式延迟，需要在发布时刷新资源版本
